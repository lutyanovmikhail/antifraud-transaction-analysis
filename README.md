# antifraud-transaction-analysis
Anti-Fraud Prototype with NLP & Behavioral Features
ОПИСАНИЕ ПРОЕКТА
Проект демонстрирует прототип антифрод-пайплайна для выявления мошеннических транзакций с использованием:
поведенческих числовых признаков,
базового feature engineering,
модели логистической регрессии,
текстовых сигналов (NLP), потенциально получаемых из OSINT-источников.
Данные в проекте ненастоящие и используются исключительно для демонстрации подхода и логики.
Постановка задачи
Цель — классифицировать транзакции на:
0 — легитимная транзакция
1 — мошенническая транзакция
Основной фокус — поиск фрода, где стоимость пропущенной мошеннической операции  выше, чем ложнопозитивного срабатывания .
Используемые данные
ненастоящий датасет (sklearn.make_classification)
10 числовых признаков, имитирующих характеристики транзакций
Целевая переменная is_fraud
Дополнительно используются короткие текстовые сообщения, имитирующие:
описания транзакций
подозрительные комментарии
Feature Engineering
В проекте реализованы примеры базовых антифрод-признаков:
агрегаты (суммы признаков),
отношения признаков,
признаки, отражающие изменение поведения.
МОДЕЛЬ
Используется логистическая регрессия как:
модель, часто применяемая в антифроде на раннем этапе.
Оценка производится на выборке
NLP-компонент
Текстовые данные обрабатываются с помощью CountVectorizer.
NLP рассматривается как дополнительный слабый сигнал
использоваться совместно с числовыми признаками.
Ограничения проекта
Используются ненастоящие данные
Не учитывается сильный классовый дисбаланс
Нет временной динамики транзакций
NLP-компонент представлен в упрощённом виде
Возможные улучшения
Работа с реальными или более реалистичными данными
Добавление временных окон
Балансировка классов
Использование более сложных NLP-моделей
Оптимизация модели под реколл
